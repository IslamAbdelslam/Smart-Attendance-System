## 🔍 Overview

This system automatically marks student attendance using face recognition from a camera feed. Students are recognized as they enter the classroom — no manual roll call needed.

**Key Features:**
- Detects multiple faces simultaneously in one frame
- Never confuses two different people (0 wrong predictions)
- Logs attendance once per student per session (anti-duplicate)
- Works with laptop webcam or any IoT camera via WiFi
- Enrolls new students in under 30 seconds without retraining

---

## 🔄 Pipeline

```
Phase 1 — Offline Preprocessing (Google Colab)
──────────────────────────────────────────────
Raw Photos
    ↓
Data Cleaning (HEIC conversion, format fixes)
    ↓
YOLOv8s-Face → Detect faces
    ↓
insightface → Align faces to 112×112
    ↓
Data Augmentation → expand to 100 images/person
    ↓
ArcFace (w600k_mbf) → Extract 512-dim embeddings
    ↓
Average embeddings per person → Save database.pkl

Phase 2 — Real-Time Attendance (VS Code / Python)
──────────────────────────────────────────────────
Camera Frame
    ↓
YOLOv8s-Face → Detect all faces simultaneously
    ↓
Crop + Resize to 112×112
    ↓
ArcFace → Extract embedding
    ↓
Cosine Similarity vs database.pkl
    ↓
Mark present + Log to CSV
```

## ⚙️ Installation

### Requirements
- Python 3.11 (recommended — 3.14 has library compatibility issues)
- Windows / Mac / Linux
- Webcam or IoT camera

### Step 1 — Clone Repository
```bash
git clone https://github.com/yourusername/smart-attendance-system.git
cd smart-attendance-system
```

### Step 2 — Create Virtual Environment
```bash
# Create venv with Python 3.11
py -3.11 -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (Mac/Linux)
source venv/bin/activate
```

### Step 3 — Install Dependencies
```bash
pip install -r requirements.txt
```

## 🚀 Usage

### Run Real-Time Attendance
```bash
python main.py
```
- Camera opens automatically
- All detected faces are recognized simultaneously
- Attendance logged to `logs/attendance_log.csv`
- Press **Q** to quit and see session summary

### Enroll New Student
```bash
python enroll.py
```
- Enter student name
- Press **SPACE** 5 times to capture face
- Student added to database instantly
- No retraining required

### Switch to IoT Camera
Edit `config.py`:
```python
# Laptop webcam (default)
CAMERA_SOURCE = 0

# ESP32-CAM via WiFi
CAMERA_SOURCE = "http://192.168.1.100/video"

# IP Camera via RTSP
CAMERA_SOURCE = "rtsp://192.168.1.100:554/stream"
```

## 🔧 Configuration

All settings are in `config.py`:

```python
THRESHOLD      = 0.3     # Recognition confidence (0.3 = best accuracy)
CAMERA_SOURCE  = 0       # Camera source
PADDING        = 20      # Pixels around detected face
IMG_SIZE       = 112     # ArcFace standard input size
DETECTION_CONF = 0.5     # Minimum YOLO confidence
```

## 🛠️ Technologies Used

| Component | Technology | Purpose |
|-----------|-----------|---------|
| Face Detection | YOLOv8s-Face (lindevs) | Detect faces in frame |
| Face Alignment | insightface buffalo_sc | Align to 112×112 (preprocessing) |
| Face Embedding | ArcFace w600k_mbf | Extract 512-dim identity vector |
| Similarity | Cosine Similarity | Match query vs database |
| Camera | OpenCV VideoCapture | Read camera frames |
| Runtime | ONNX Runtime | Run .onnx models on CPU |

### Key Papers
- **YOLOv8**: Jocher et al. (2023). Ultralytics YOLO v8.0.0
- **ArcFace**: Deng et al. (2019). CVPR 2019. [arxiv.org/abs/1801.07698](https://arxiv.org/abs/1801.07698)
- **RetinaFace**: Deng et al. (2020). CVPR 2020
- **LFW Dataset**: Huang et al. (2007). University of Massachusetts
